## Q3(pdf Q#101)

솔루션 아키텍트는 프라이빗 서브넷에 인터넷 액세스를 활성화하기 위해 무엇을 해야 할까요?

B. 각 AZ의 프라이빗 서브넷마다 하나씩, 총 세 개의 NAT 인스턴스를 생성합니다. 각 AZ에 대해 VPC 트래픽이 아닌 트래픽을 해당 AZ의 NAT 인스턴스로 전달하는 프라이빗 라우팅 테이블을 생성합니다.
C. 프라이빗 서브넷 중 하나에 두 번째 인터넷 게이트웨이를 생성합니다. VPC 트래픽이 아닌 트래픽을 프라이빗 인터넷 게이트웨이로 포워딩하도록 프라이빗 서브넷의 라우팅 테이블을 업데이트합니다.
D. 퍼블릭 서브넷 중 하나에 송신 전용 인터넷 게이트웨이를 생성합니다. VPC 트래픽이 아닌 트래픽을 송신 전용 인터넷 게이트웨이로 포워딩하도록 프라이빗 서브넷의 라우팅 테이블을 업데이트합니다.
